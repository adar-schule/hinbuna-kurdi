Hinbuna Kurdi (Hinbûna Kurdî) is a Kurdish language learning platform teaching the Kurmanji dialect at A1 through B1 proficiency levels. It targets the Kurdish diaspora and anyone interested in learning Kurdish through a structured, curriculum-based approach.

Tech Stack

  • Backend: NestJS (Node.js + TypeScript)
  • Frontend: React + Tailwind CSS
  • Database: PostgreSQL (AWS RDS)
  • Auth: JWT + Passport.js
  • Hosting: AWS (ECS + RDS + S3)
  • TTS/STT: kurdishtts.com API

Current Phase

We are in the Design Phase — creating screen mockups as pure, self-contained HTML/CSS files. These will later become the blueprint for the React + Tailwind implementation. No backend or frontend code is needed right now.

Team

  • Armanc (Lead) — Backend, DevOps, Architecture, Design decisions
  • Gule (Frontend) — UI/UX, React implementation
  • Mamoste (Content) — Kurdish curriculum, lessons, activities

Scope

There are 32 screens total, across 4 user roles:

  • Public (P1–P6) — Landing, Pricing, Login, Register, Forgot Password, About
  • Student (S1–S12) — Dashboard, Courses, Modules, Lessons, Activities, Results, Profile, etc.
  • Teacher (T1–T8) — Content Dashboard, Editors, Materials, Student management
  • Admin (A1–A6) — Platform Dashboard, User Mgmt, Subscriptions, Settings

Check the Design Gallery for the full list with current status.

Claude Code is our primary development tool for this project. All screen designs, code, and documentation are created through Claude Code in the terminal.

Context limit: Claude Code sessions have a context window. When it reaches ~70%, save your work and start a new session. Claude will read the status files and pick up where you left off.

Follow these steps for every new screen. This ensures consistency across all 32 screens.

  1. 1

    Check the Design Gallery

    Open index.html and find the next screen with "Pending" status. Work in priority order (P0 first, then P1, then P2).

  2. 2

    Read the Screen's Prompt File

    Open _docs/prompts/[screen].md — for example, _docs/prompts/P2-pricing.md. This contains the agreed-upon layout sections, content structure, and screen-specific notes.

  3. 3

    Read the Base Template

    Open _docs/prompts/00-base-template.md. This is the complete brand system reference: colors, fonts, component patterns, anti-patterns, and quality checklist. Every screen must follow this.

  4. 4

    Invoke the frontend-design Skill (if using Claude Code)

    Before generating, invoke the frontend-design skill. This ensures production-grade quality, intentional spatial composition, and meaningful micro-interactions.

  5. 5

    Generate LIGHT MODE HTML First

    Create the light mode version and save it as site/[screen]-light.html. For example: site/P2-pricing-light.html.

  6. 6

    Review in Browser at 375px

    Open in Chrome at 375px width. Check colors, typography, spacing, visual rhythm. Iterate until it meets the quality bar. Refer to the checklist in Section 8.

  7. 7

    Generate DARK MODE HTML

    Create the dark mode version: same layout, same content, only colors change. Save as site/[screen]-dark.html. See Section 4 for the exact dark mode color mapping.

  8. 8

    Update the Gallery

    In site/index.html, find the screen card and change its status from "Pending" to "Done". Remove the disabled class from the Light/Dark links so they become clickable.

  9. 9

    Commit & Push

    Follow the Git workflow in Section 7. Once pushed, the GitHub Actions workflow auto-deploys the site/ folder to GitHub Pages.

Every HTML file must be self-contained. Single file, all CSS in a <style> block. The only external dependency allowed is Google Fonts (Nunito + Inter).

NEVER reference Duolingo — not in prompts, designs, code, comments, or comparisons. This is a serious, structured language learning platform. Not gamified, not playful, not cartoon-style.

Brand Vibe

Calm + SaaS — think Headspace meets Notion. Premium, sophisticated, warm botanical.

  • YES: Professional hierarchy, trust signals, structured curriculum feel, clean typography, warm tones
  • NO: Gamified elements, playful animations, cartoon mascots, childish iconography, neon colors

Color System — Light Mode

Token Hex Usage
Background #FDF8F3 Page background — warm cream
Surface #F5EEE6 Card backgrounds, inputs — soft beige
Primary #2D5A3D Buttons, links, headings — forest green
Accent #D4A843 Badges, highlights, CTAs — warm gold
Tan #E3C79D Secondary surface, subtle accents
Text #1A2A1F Body text — dark green-black
Text Secondary #5A6B5E Subtitles, hints — muted green-gray
Border #E5DDD3 Card borders, dividers — warm tan
Success #2D8A4E Correct answers, completed items
Error #DC4545 Validation errors, wrong answers
Warning #EFB034 Caution states, expiring items
Info #3B7A8A Informational banners, hints

Color System — Dark Mode

Token Hex Usage
Background #1A2024 Page background — deep forest green (NEVER pure black)
Surface #232D31 Card backgrounds, inputs — slightly lighter green
Primary #2D5A3D Links, structural elements — forest green
Accent #D4A843 Primary CTAs, badges — warm gold (brightest element)
Text #F5EEE6 Body text — warm off-white
Text Secondary #9CADA8 Subtitles, hints — soft sage
Border #2F3B3E Card borders, dividers — subtle green

Dark mode rule: Same layout as light, only colors change. Dark background = #1A2024 (deep forest green). The feeling is "warm forest at night" — NOT matrix, NOT neon, NOT hacker terminal. Gold accents glow like candlelight against deep green.

Typography

Hero / H1 Hinbûna Kurdî Nunito 700 / 28–32px
Heading / H2 Section Title Nunito 600 / 22–24px
Subhead / H3 Card Heading Nunito 600 / 16–18px
Body Regular body paragraph text Inter 400 / 14–16px
Link Clickable link text Inter 500 / 14–16px
Caption Helper text, labels, small print Inter 400 / 12–13px
Overline CATEGORY LABEL Inter 600 / 11–12px uppercase

Component Patterns

Buttons:

  • Rounded corners (12px), full-width on mobile, 44–48px height
  • Light mode: forest green #2D5A3D bg + cream text
  • Dark mode: gold #D4A843 bg + dark text
  • Secondary: outlined with matching border color

Cards:

  • Rounded corners (12–16px), subtle shadow
  • 1–2 shade difference from background — not harsh borders
  • Light: beige #F5EEE6 on cream / Dark: #232D31 on dark bg

Inputs:

  • Rounded (8–12px), icon on the left side
  • Light: beige fill, green border on focus
  • Dark: dark surface fill, cream border on focus

Anti-Patterns — NEVER Do These

  • No pure black backgrounds — use deep forest green #1A2024
  • No neon or electric green — use forest green #2D5A3D
  • No bright white text on dark — use warm cream #F5EEE6
  • No harsh card borders — use subtle shade difference
  • No cold blue-gray neutrals — use green-tinted warm grays
  • No flag icons for languages — use monochrome text labels
  • No gamified or cartoon elements — keep professional and calm

Run through this checklist after every screen before committing. Click each item to mark it checked (for your own tracking — state is not saved).

  • Background color correct? Light: cream #FDF8F3 / Dark: deep forest #1A2024
  • NOT matrix green? Dark mode = warm forest at night, not hacker terminal
  • Gold accents present? At least one gold element #D4A843 on the screen
  • Text readable? Sufficient contrast between text and background
  • Buttons correct? Green + cream (light) / Gold + dark (dark)
  • Cards have subtle separation? Shade difference, not harsh borders
  • Typography correct? Nunito for headings, Inter for body text
  • Kurdish text correct? Characters ê, î, û, ç, ş display properly
  • Mobile width? 375px base, responsive to wider screens
  • Header consistent? Logo left, globe + toggle + action right
  • No flag icons? Language picker uses text labels, monochrome
  • Sections flow naturally? Visual rhythm, not stacked blocks with equal gaps
  • Touch targets 44px+? All interactive elements large enough to tap
  • CSS animations present? Hover states, fade-ins, smooth transitions
  • Self-contained? Single HTML file, only Google Fonts as external dependency

If all 15 items pass, your screen is ready to commit. If any item fails, fix it before pushing. Consistency across all 32 screens is what makes the design system work.
